Directive definition object properties

Notice: All directive examples are defined and located in /directives/directive_examples/ folder and are put in directives.html file.

TEMPLATE

  1. template: 'html string' - template used as a HTML string [01template-string.js]
  2. template: function(tElement, tAttrs) { ... } - template used as a function which returns HTML string [01template-func.js]
  3. templateUrl: 'mytemplate.html' - template URL is using HTML file in /dist/ folder [02templateurl.js]
  4. templateUrl: function(tElement, tAttrs) { ... } - template used as a function which returns template URL [02templateurl-func.js]
    NOTICE: In this example template is defined inside JS file (main.js) using $templateCache. 
    //define default templates
clientApp.run(function ($templateCache) {
    'use strict';
    $templateCache.put('green.html', '<p style="color:green">green template</p>');
    $templateCache.put('red.html', '<p style="color:red">red template</p>');
});
  5. templateNamespace: 'html | svg | math' - document type representation ('html' is default) [02templatenamespace.js]

CONTROLLER

  1. controller: 'someCtrl' - external controller is used [03controller-external.js, directivesCtrl.js]
  2. controller: function($scope) { ... } - internal controller [03controller-internal.js]
    NOTICE: $scope which is outside template can be used inside template with {{myVar}}.
  3. controller: function($scope, $element, $attrs, $transclude, otherInjectables) { ... } - usage of $element, $attrs and $transclude [03controller-internal.js]

SCOPE

Define scope inside directive. Possible values:
  1. scope: false - No new scope. The controller and directive are using the same scope object. This means any changes to the controller or directive will be in sync.
    Usually scope is defined inside controller: property. false is default value. [05scope-false.js]
    IMPORTANT: A value inserted in 'childScopeVar' input will appear in parent scope.

    childScopeVar in parent scope: {{childScopeVar}}

  2. scope: true - New child scope will be created inside directive. Any changes made to this new scope will not reflect back to the parent scope. [05scope-true.js]
    IMPORTANT: A value inserted in 'childScopeVar' input will NOT appear in parent scope. That's because true creates new child scope which is different from parent scope.
    If you want that variable to be accessible in parent scope put ng-model="$parent.childScopeVar2"

    childScopeVar2 in parent scope: {{childScopeVar2}}

  3. scope: {isoScope: '@attr'} - New isolated scope for reusable angular directives. @ binds 'isoScope' to directive's attribute. [05scope-object-at.js]
    IMPORTANT: On the contrary to true, object {} creates new isolated scope, so that scope doesn't inherit anything from the parents.

    isoScopeVar in parent scope: {{isoScopeVar}}

  4. scope: {isoScope: '=attr'} - New isolated scope. = binds 'isoScope' to directive's attribute where model is defined. It is bidirectional binding. [05scope-object-eq.js]
    IMPOTRANT: = is bidirectional binding what means changes in 'parentModel' will reflect to 'localModel' and vice versa. (Try to input something into 'localModel' input field.)
    parentModel in parent scope: {{parentModel}}

OTHER PROPERTIES

  1. restrict: 'E | A | C | M' - E-element, A-attribute, C-class, M-comment (EA in most cases)
    replace: true | false - will be directive element replaced with real HTML tag (default is false)
    [04restrict-e.js, 04restrict-a.js] 
    //template-- <p style="color:green"> </p>
//directive--  <mydir class="pull-right"> </mydir>

//replace: true (will merge directive and template attributes)
//result will be:  <p class="pull-right" style="color:green"> </p>


//replace: false
<mydir class="pull-right"> <p style="color:green"> </p> </mydir>
